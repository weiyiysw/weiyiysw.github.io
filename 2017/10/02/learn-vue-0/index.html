learn-vue-0

  • vue, development

工欲善其事，必先利其器

在开始学习一门技术之前，我们最先要做的就是搭建好一个开发环境。这系列文章是有关于学习vue.js的，关于什么是vue.js，读者可以自行百度。接下来，开始搭建开发环境。

搭建基础环境

1. 安装Node.js

关于Node.js的安装这里不多叙述，大家在官网下载对应系统的安装包即可。

注意：目前新版的Node.js都已经包含了npm，无需额外安装。

2. 使用cnpm

在命令行下执行以下命令：

npm install -g cnpm --registry=https://registry.npm.taobao.org

成功之后就可以使用cnpm代替npm。

因为在国内使用npm经常会很慢，因此选择使用cnpm。这是淘宝的镜像源，速度会快很多。
cnpm镜像网址

3. package.json

这是一个JSON配置文件，管理你项目开发的依赖。如果你是一个Java开发者，那么你肯定用过maven或gradle管理项目的依赖。这个package.json文件也是如此。

package.json必须包含两个字段：“name”、“version”.。还有很多其他的字段，大家可以自行查阅相关文档。

在项目中，规定此项目的依赖包，在文件中可以定义为两种类型：“dependencies”、“devDependencies”。前者是在生成环境中需要依赖的包，后者仅在开发和测试环境需要的包。

大家可以通过手动在package.json文件中添加，但更佳的方式是使用npm install的–save和–save-dev命令。前者将依赖写入到“dependencies”中，后者写入到“devDependencies”。

这样，其他人使用你的项目，只需要执行命令npm install即可。

4. webpack

webpack是现代JavaScript开发的模块打包工具。webpack中文网站大家更详细的信息可以访问这个网址学习webpack。

这里简要记录webpack的四个核心概念：

  • 入口（Entry）：告诉webpack应该从哪个文件开始
  • 出口（Output）：将所有资源获取到之后，告诉webpack在哪里打包应用。
  • Loader：webpack自身只理解JavaScript，并且webpack将每个文件（.css,.html,.scss,.jpg,etc）都作为模块处理，Loader就是将这些转换为模块。
  • 插件（Plugins）：由于 loader 仅在每个文件的基础上执行转换，而 插件(plugins) 更常用于（但不限于）在打包模块的 “compilation” 和 “chunk” 生命周期执行操作和自定义功能。

第一个Vue应用

Vue.js官网不推荐新手从vue-cli入手第一个应用，因为vue-cli是一个Vue的脚手架，直接上手新手可能会对Vue的基本语法不熟悉。然而，这里将会直接从脚手架开始，逐步的介绍一些Vue.js的语法。我直接查看官网的文档，只是对Vue.js的语法有个初步的了解，没有更深入的理解。我觉得从脚手架开始，是一个宏观上的学习方式，到具体某一个细节时，会对Vue.js有更加深入的理解。


UI

选好一套UI框架，并依此进行开发，能够让开发的应用更加美观。本篇文章中选择了element-ui——由饿了么团队开发维护的。

第一个应用

打开GitHub找到vue-cli项目，这是vue-cli链接。

以下步骤在此项目下，均能找到。下面，复述记录一遍。

首先，全局安装vue-cli。

npm install -g vue-cli

然后，使用vue init <template-name> <project-name>即可生成项目。例如：执行vue init webpack first-vue，按照提示输入确认即可。

目前提供的template有以下几个：

  • webpack
  • webpack-simple
  • browserify
  • browserify-simple
  • pwa
  • simple

生成了项目之后，执行npm install安装依赖。由于国内网速问题，建议使用cnpm install安装依赖，速度将会很快。安装完成后，执行npm run dev，即可成功启动项目，并会自动打开浏览器访问http://localhost:8080。

至此，第一个应用已经成功执行。

尾语
