learn-vue-1-分析项目结构

  • vue, vue-cli

项目结构

first-vue项目下文件结构如下（未列出隐藏文件）：

文件

  • README.md：markdown文档，通常为项目的说明文档
  • index.html：应用的根页面
  • package.json：管理项目依赖

文件夹

  • build：构建的配置文件
  • config：配置文件
  • node_modules：本项目依赖下载完成后存储仓库
  • src：源码目录
  • static：静态页面文件夹，存储其他静态页面

src目录下文件结构如下（未列出隐藏文件）：

文件

  • App.vue：应用根组件
  • main.js：Vue应用入口

文件夹

  • assets：资源文件夹
  • components：vue组件
  • router：vue路由

一些语法

main.js里语法分析

语法意义：创建一个Vue实例。

和面向对象语言（如：Java）新建一个对象类似。我们可以看到，new语法后跟了一个大括号，里面有四个语句。

  • el：类型是HTMLElement，即Vue实例使用的根DOM元素。（详见vue的API文档）
  • router：是路由，查看详细的main.js文件夹可以看到，这是由import语句导入的，稍后分析。
  • template：一个字符串模板作为 Vue 实例的标识使用。模板将会 替换 挂载的元素。挂载元素的内容都将被忽略，除非模板的内容有分发插槽。（详见vue的API文档）
  • components：组件，将Vue实例使用的组件放到这里。

阅读分析生成的第一个Vue实例代码时，结合Vue官网对Vue的介绍以及Vue的API，更容易理解。

单文件组件——App.vue

我们在main.js里的template可以看到值为<App/>。第一次看到时，可能会疑惑，这是什么呢？从main.js的import语句，我们可以看到import App from './App'。我们查看目录，可以看到目录下有个App.vue文件。

这里，我们需要学习到Vue一个重要的知识点——单文件组件。文件扩展名为.vue是Vue的单文件组件。以下内容是Vue官网对于单文件组件的介绍：

在很多 Vue 项目中，我们使用 Vue.component 来定义全局组件，紧接着用 new Vue({ el: ‘#container ‘}) 在每个页面内指定一个容器元素。
这种方式在很多中小规模的项目中运作的很好，在这些项目里 JavaScript 只被用来加强特定的视图。但当在更复杂的项目中，或者你的前端完全由 JavaScript 驱动的时候，下面这些缺点将变得非常明显：
全局定义 (Global definitions) 强制要求每个 component 中的命名不得重复
字符串模板 (String templates) 缺乏语法高亮，在 HTML 有多行的时候，需要用到丑陋的
不支持 CSS (No CSS support) 意味着当 HTML 和 JavaScript 组件化时，CSS 明显被遗漏
没有构建步骤 (No build step) 限制只能使用 HTML 和 ES5 JavaScript, 而不能使用预处理器，如 Pug (formerly Jade) 和 Babel
文件扩展名为 .vue 的 single-file components(单文件组件) 为以上所有问题提供了解决方法，并且还可以使用 Webpack 或 Browserify 等构建工具。

路由

这里我们将解答一个疑问，App.vue里的组件是如何找到Hello.vue并替换。我们打开router文件夹下的index.js文件，里面新建了一个路由对象。

在路由对象里，有一个routes属性，定义了路径为“/”，则转到组件Hello。也就是说，当页面上访问http://localhost:8080/时，访问到了主页面App.vue内容，由于路径是“/”，所以router-view组件将由Hello组件替换。
